Defines BanManager's messages throughout the plugin.

Messages support [JSON](https://minecraft.fandom.com/wiki/Raw_JSON_text_format) and [Minecraft colour codes](https://gamedb.gg/games/minecraft/bukkit-color-codes/) using the `&` delimiter. Use `\n` for new lines. Text wrapped inside square brackets i.e. `[player]` are tokens which will be replaced with their corresponding value. Set values to an empty string, i.e. `''` if you do not wish the message to display.

| Platform | Location |
| -------- | -------- |
| CraftBukkit/Spigot/Paper | `/plugins/BanManager/messages.yml` |
| BungeeCord | `/plugins/BanManager/messages.yml` |
| Sponge | `/config/banmanager/messages.yml` |
| Velocity | `/plugins/BanManager/messages.yml` |

- ###### `sender`
  A list of generic messages used by most commands for common scenarios.
  - ###### `error`
    - ###### `notFound`
      Displayed when attempting to perform a command on a player which cannot be found in BanManager's database
      <Message name="sender.error.notFound" />
    - ###### `offline`
      Displayed when attempting to perform a command which requires a player to be online, e.g. `/kick <player> <reason>`
      <Message name="sender.error.offline" />
    - ###### `noSelf`
      Displayed when attempting to perform a command on yourself, e.g. banning yourself
      <Message name="sender.error.noSelf" />
    - ###### `exception`
      Displayed when attempting an unknown error occurs, check your server console &amp; logs for further information.
      <Message name="sender.error.exception" />
    - ###### `invalidIp`
      Displayed when attempting to perform a command with an incorrectly formatted IP address, e.g. `/banip 127.0.1`
      <Message name="sender.error.invalidIp" />
    - ###### `ipNotSupported`
      Displayed when attempting to perform a command which does not support IP addresses, e.g. `/bminfo 127.0.0.1`
      <Message name="sender.error.ipNotSupported" />
    - ###### `offlinePermission`
      Displayed when attempting to perform a command on an offline player without the neccessary offline permission, e.g. [bm.command.ban.offline](/docs/banmanager/permissions?search=offline)
      <Message name="sender.error.offlinePermission" />
    - ###### `exempt`
      Displayed when attempting to perform a command on a [player who is exempt](/docs/banmanager/configuration/exemptions-yml)
      <Message name="sender.error.exempt" />
    - ###### `noPermission`
      Displayed when attempting to perform a command without the neccessary permissions, such as attempting to use a command flag e.g. `/ban -silent <player> <reason>`
      <Message name="sender.error.noPermission" />
    - ###### `invalidReason`
      Displayed when attempting to use a reason shortcut which does not exist, e.g. `/ban confuser #doesnotexist`, requires enabling [blockInvalidReasons](/docs/banmanager/config-yaml#blockinvalidreasons)
      <Message name="sender.error.invalidReason" />
